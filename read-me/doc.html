Illustrator

O layout deve ser gerado no Illustrator prioritariamente. Existe um arquivo modelo pronto para isso. Neste arquivo existem layers prontas com os elementos além das paletas igual à usada no sistema atual.

As cores da paleta são globais e alteram os elementos automaticamente. Para criar uma nova paleta siga o mesmo padrão, inclusive de nome, para que possa ser transcrito no arquivo de configuração do Front-Alego.

Quando alterar ou criar uma nova paleta, comece pela quarta cor (a cor do meio) de cada paleta e crie as três variantes mais claras (as 3 primeiras) e as três variantes mais escuras (as 3 últimas).

Regras de padronização

BEM - Bloco, Elemento, Modificador

Estamos utilizando a regra do BEM para nomear as classes.

Bloco
O bloco é o elemento pai, principal. Procure usar nome relevante e de fácil identificação. Se precisar separar palavras do nome, use apenas um traço: -
Elemento
É o elemento filho do bloco. Ele deve ser aparecer logo depois do bloco e separado com dois sublinhados: __
Modificador
Uma variação do bloco ou elemento. Deve ser separada com dois traços: --

Exemplo

Imagine a seguinte estrutura: você tem uma caixa com um cabeçalho, um corpo com lista e formulário e um rodapé com ações.

Os nomes seriam:

  • Caixa: .box
  • Cabeçalho: .box__cabecalho ou .box__header
  • Corpo: .box__main ou .box__corpo
  • Rodapé: .box__footer ou .box__rodape.
  • Formulário: .box__form ou .box__main__form. A primeira opção é melhor, a menos que você tenha mais de um formulário dentro da caixa e ela tenha formatações diferentes.
  • Tabela: .box__tab ou .box__main__tab.

Se tivéssemos uma caixa com variação (exemplo: urgente), ela seria .box--urgente.

Se você tivesse uma tabela com os dados maior poderia ser .box__tab--maior.

Filhos pegam o nome variante do pai?

Não. Os elementos filhos fazem uso do nome do elemento pai sem o modificador e o CSS é formatado com uso de descendente:

HTML

<div class="box--urgente">

<section class="box__tab">

<...>

</section>

</div>
CSS

.box__tab { ... } /* TODOS os .box__tab */

.box--urgente { ... }

.box--urgente .box__tab { ... } /* Os box__tab que são filhos do .box--urgente

É melhor desta forma do que termos uma classe .box--urgente__tab. Primeiro porque nem sempre teremos configuração específica do elemento filho quando ele estiver dentro do pai (e teremos que extender uma classe com a outra). Segundo porque diminui a quantidade de classes usadas no sistema.

O grande defeito dessa abordagem é o uso de seletores descendentes (ver mais abaixo).

BEM no SASS

O SASS facilita a criação dos elementos filhos e variantes, inclusive com a opção de retrair e expandir o código diretamente no editor. Para isso funcionar basta usar o símbolo de ampersand (&).

SASS

.box {

&__header { ... }

&__main { ... }

&--urgente {

@extend .box;

...

}

}

Isso vai resultar em:

CSS

.box, .box--urgente { ... }

.box__header { ... }

.box__main { ... }

.box--urgente { ... }

Nomes de classes

Jamais use nas classes específicas elementos de formatação. Estes devem aparecer como classe adicional ou modificador de classe (--mod).

Exemplo: se você tiver um card que deve aparecer alinhado na esquerda da página, não use .card-alinha-esquerda e sim .card--esquerda ou ainda duas classes (.card e .float-left).

Para elementos comuns de formatação use as seguintes simplificações:

bt
botão
tab
tabela
ico
ícone, símbolo
form
formulário

Organização dos arquivos SCSS

Como explicado aqui os arquivos são armazenados em pastas específicas. Elementos gerais que fujam do padrão do Front-Alego devem ser salvos em arquivos separados dentro da pasta /extra. Um exemplo disso é tabelas. Mixins e funções também vão nesta pasta. Por outro lado, somente os placeholders e classes que são de uso global (em mais de um arquivo scss) devem vir na pasta extra.

Comentários

O arquivo deve ser totalmente comentado ao estilo SASS para identificar seus trechos.

Ele deve abrir com um comentário maior com o nome do sistema, o usuário que criou o arquivo e a data de criação. Veja como exemplo o CSS usado neste arquivo:

================================================================================

FRONT-ALEGO

--------------------------------------------------------------------------------

Arquivo de CSS modelo utilizado na documentação. Use o arquivo SCSS base desta

pasta (info.scss) para ver como foi utilizado e também para acrescentar

eventuais elementos para visualizar o arquivo final.

Não se esqueça de compilar

--------------------------------------------------------------------------------

Autor: Alessandro Guarita

Criado em: 23/11/2015

================================================================================

Cada parte do arquivo deve ser separada com comentários usando sinal de igual até aproximadamente 80 caracteres. E o nome deve ser em caixa alta. Exemplo:

// =============================================================================

// HEADER

// =============================================================================

Sub-seções são separadas da mesma forma acima, mas com uso de traço no lugar do igual. No comentário coloque o nome do elemento pai e separe com pipe (|):

// -----------------------------------------------------------------------------

// HEADER | MENU

// -----------------------------------------------------------------------------

Ordem dos elementos

Os elementos dentro de um arquivo devem obedecer a seguinte ordem (sempre comentado, conforme explicado acima):

  1. Variáveis e mapas gerais
  2. Placeholders gerais
  3. Classes gerais (que só usam neste arquivo e não são de nenhuma seção específica)
  4. Seções do site (header, menu, principal, footer)
Variáveis e placeholders podem aparecer dentro de uma seção do site no CSS, caso não se repitam em nenhum lugar. Neste caso ficaria assim para facilitar a localização e modificação.

Atualização emergencial

Você conhece a situação: você tem que fazer uma alteração pra ontem. Por mais que o objetivo do Front-Alego™ seja organizar os arquivos, isso pode levar um tempo que não se dispõe. Neste caso haverá uma quebra das regras, temporariamente. Mas mesmo quebrando regras, temos regras, porque aqui não é a Casa da Mãe Joana. Siga os passos:

  1. Crie um arquivo chamado ajustes.scss na pasta extras
  2. Escreva um comentário indicando para quê é essa alteração e seu nome
  3. Escreva as regras. Não tenha medo de usar !important e outras regras escrotas pouco apropriadas
  4. No final do arquivo principal do seu CSS que a regra tem que ser aplicada dê um import para o arquivo:

    @import "extras/ajustes"
  5. Teste
  6. Commite o trabalho
  7. Depois que a corda saiu do seu pescoço insira as alterações nos lugares corretos e com as regras de boas condutas aplicadas
  8. Delete o @import para o arquivo de ajustes (pode deletar o conteúdo dos ajustes também)

Seletores descendentes e aninhamento de código

Evite o máximo possível CSS com seletor descendente (.classe1 .classe2). Este seletor diminui a performance do browser e aumenta a especifidade do código. Por esta mesma razão evite grandes aninhamentos de código no SASS. Esta regra, no entanto, tem exceções (uma delas já mostrada acima, no BEM). Mas mesmo assim, tenha no máximo dois descendentes e aninhamento (exemplo: .tab td e não .tab tr td).

Medidas relativas

Para que o sistema seja adaptável, sempre use medidas relativas. Jamais use valores em pixels (a exceção é tamanho de imagens e vídeos externos). Para os demais casos use em, rem, %, vw, etc.

REM vs. EM

Sabendo da diferença entre ambos (rem é relativo ao tamanho da fonte do root e em é relativo ao tamanho da fonte do elemento pai), use os dois quando adequado. Dicas:

  • Tamanho da fonte que é para ser relativa ao tamanho pai, use em. Exemplo: um link, um destaque ou mesmo todo um bloco de textos.
  • Tamanho de fonte que é para se manter com o mesmo tamanho, independente do tamanho do elemento pai, use rem. Exemplo: título da página.
  • Margens costumam ser mais adequado utilizar rem para manter a consistência.
  • Paddings, por outro lado, é mais adequado usar em para que o espaço interno cresça conforme se aumenta a fonte. Exceção: se você tem um elemento com dois tamanho de fonte diferentes, manter o padding em rem ajuda a manter a consistência. (exemplo: tabela)
Existem funções no bourbon que convertem o valor de pixels para rem ou em. Basta usar a função rem() ou em()

Media Queries

O uso de Media queries para fazer sites responsivos não usa medidas em pixels mais sim em rem, pois assim se adapta inclusive ao zoom dado pelo usuário.

Ver mais detalhes sobre isso neste artigo.

A ideia básica é parar de usar medidas de dispositivos específicos (que a cada momento tem novos) e trabalhar com a medida da tela em si.

Sempre use o sistema de media-query descrito aqui (fazendo uso da função breakpoint). E nunca, jamais, escreva uma medida que não esteja descrita no mapa $breakpoints.

Ordem das propriedades dentro de um seletor

Calma, não precisa organizar TODOS as propriedades com uma regra específica. Mas as propriedades devem seguir a seguinte regra básica:

  1. Primeiro aparecem os @extend (seja para um placeholder ou para outra classe). Isso é importante pois você pode incluir uma propriedade que altera o que você extendeu.
  2. Depois aparecem os @include para os mixins
  3. Se você duplicar regras para teste (exemplo: dois background diferentes) lembre-se de comentar a que não está sendo usada ou, melhor ainda, deletar.

Cores

O Front-Alego™ trabalha com as cores de duas formas: através de uma paleta de cores que pode ser chamadas por nomes específicos (exemplo básica) e através de um tema que pode alterar todas as cores do sistema, sem precisar mudar a paleta, mantendo a harmonia.

As cores ganham variações clara e escura através de uma função baseada no Bourbon.

Todas as cores devem ser definidas aqui. Jamais chame uma cor em outro CSS usando hexadecimal, nome de cor ou variável!

Paletas

Neste arquivo você irá configurar as cores usadas pelo sistema. Atualmente temos duas cores principais (basica e secundaria), mas você pode acrescentar conforme a necessidade. As demais cores são para temas específicos. Você pode alterar, mas tome cuidado.

Cada cor representa uma matiz principal e deve ter um nome fácil de lembrar, com um comentário indicando qual a cor (apenas para facilitar a lembrança, é possível ver todas as paletas aqui). Sempre comece alterando as já estabelecidas e acrescente novas conforme a necessidade.

Segue a lista indicativa de matizes criadas até o momento:

basica
A cor básica/principal do sistema. Altere esta no seu sistema como cor principal.
secundaria
A cor secundária/destaque do sistema. Altere esta cor no seu sistema como cor secundária.

Inserindo cores dentro do tema

Para chamar as cores dentro dos temas, vai depender se ela vem da $paleta que você configurou aqui ou se faz parte da $paleta-padrao. Cada qual usa uma função SASS, conforme listado abaixo:

paleta-padrao()
Use para chamar as cores da $paleta-padrao. Exemplo: para definir uma cor como 50% de cinza, use paleta-padrao(cinza-50)
paleta()
Esta função é para chamar as cores que você criou na $paleta. Para chamar a cor pura, escreva paleta(nome-cor). Para chamar as variações claras, chame com paleta(nome-cor, clara, $valor). O $valor pode ir de 1 a 100. Para chamar cores escuras chame com paleta(nome-cor, escura, $valor)

Inserindo as cores no SASS

Como explicado anteriormente, você só poderá usar as cores definidas nos temas. Para chamar uma cor, use uma das duas funções abaixo:

cor-status()
Para chamar uma das cores de status, conforme definido anteriormente. Não use para formatação de interface, somente use para reais status. Exemplo: você quer alertar que o campo preenchido está com erro. color: cor-status(erro)
cor()
Principal função para chamar as cores na interface. Cada elemento tem um conjunto de cores a ser chamado, além dos genéricos (conforme já explicado anteriormente). Exemplo: seu texto vai ser um texto destaque. cor(texto, destaque). A cor padrao de cada conjunto não precisa ser chamada, ou seja, o texto padrão seria: cor(texto).

Grid

O sistema de grid foi pensando para escrever a menor quantidade possível de classes no CSS final e também fácil de mudar a quantidade de grids, sem ficar percorrendo o HTML à procura de classes de grid. As grids foram compostas com flexbox. Mesmo que você não utilize as grids, pode fazer uso das classes de .container.

Este sistema está sendo depreciado e está aqui para retrocompatibilidade. O flexbox possibilita a formatação bem mais simples disso. Para centralizar o conteúdo (tarefa destinada ao .container) basta usar padding e assim evitamos uma <div> desnecessária.

Containers

Estas classes são usadas em algum elemento pai (normalmente <div>) para centralizar o conteúdo e determinar que ele é flex. Ele também pode ser usado mesmo que os elementos filhos não façam uso do sistema de grid.

Determinando quantas grids o elemento ocupa

Em um sistema de grid tradicional você insere uma classe grid-x no elemento que precisa ocupar "x" medidas de grid. Ao invés de trabalhar desta forma, cada elemento receberá um nome de classe conforme explicado seguindo as regras.

Depois disso coloque a classe dentro do mapa $grid localizado dentro do arquivo config/+_config e especifique a quantidade de grids que ela ocuparia.

Como exemplo, imagine um layout simples com um menu, uma área principal e uma coluna lateral. O menu ocuparia 2 grids, a área principal 7 grids e a coluna lateral ocuparia 3 grids. Normalmente o código seria:

HTML - grid tradicional

<nav class="menu grid-2"> ... </nav>

<div class="principal grid-7"> ... </div>

<aside class="lateral grid-3"> ... </aside>

Com o sistema de grid usado no Front-Alego, ficaria:

HTML

<nav class="menu"> ... </nav>

<div class="principal"> ... </div>

<aside class="lateral"> ... </aside>
SASS

$grid: (

menu: 2,

principal: 7,

lateral: 3,

)

Ao compiltar o CSS, o SASS vai aplicar os placeholders em cada uma das classes.

z-index

config/z-index.scss

Nunca use z-index diretamente no CSS, pois isso deixa a manutenção ruim, além de usarmos números aleatórios (como o clássico z-index: 99999;).

No arquivo z-index você vai inserir os elementos com os nomes dele. Para isso inclua os elementos no mapa $z. A ordem que os elementos aparecem é o valor do z-index. O primeiro elemento vai receber z-index: 1 e assim por diante. O mapa já vem com alguns elementos pensando nessa ordem:

SASS

$z: (

conteudo,

rodape,

topo,

modal,

toaster,

);

No entanto, colocar estes elementos não basta, você deve indicar no seletor qual o nível que ele ocupa. Para isso, chama através da função z(). Dentro do parênteses você coloca o mesmo nome do elemento do mapa $z. Por exemplo:

SASS

.header {

...

z(topo);

...

}

Se você olhar no mapa $z, vai perceber que topo é o terceiro elemento, o que resulta no seguinte CSS:

CSS

.header {

...

z-index: 3;

...

}

Sendo responsivo

Sempre que for chamar uma media query, você não o fará da maneira tradicional. Para chamar uma media query você vai chamar através da função breakpoint()

breakpoint()
Procura o valor de medida no mapa $breakpoints para retornar um valor para a media query.

O mapa $breakpoints contém os valores a serem usados dentro da media-query, que são:

pequeno
Atinge dispositivos de tela pequena, como smartphones.
medio
Atinge a medida média, que pode ser tablet em pé (retrato) ou smartphone na horizontal.
grande
Pega de telas de tablet na horizontal para cima.
extra-grande
Pega medidas de notebook para cima.

Quando você especifica min-width você vai estar selecionando um valor maior que aquele. Portanto, min-width: pequeno vai pegar acima da tela pequena (mas ela não está incluída), assim como min-width: medio vai pegar telas maiores que as médias.

Quando você especifica um valor com a de max-width ele vai pegar daquela medida para baixo. Por exemplo max-width: pequeno vai pegar somente telas pequenas, enquanto max-width: medio vai pegar de tamanho médio para baixo.

Para chamar a media query você pode digitar @media da maneira tradicional ou utilizar o mixin media-tipo:

media-tipo(medida)
Este mixin chama a media query pensando na técnica mobile first (utilizando min-width). A medida é uma das descritas na lista acima.

Como trabalhar Media Query

No caso de mobile first basta usar o mixin:

SASS

.classe {

...

@include media-tipo(pequeno) {

...

}

}

Se preferir não trabalhar com o mixin, basta escrever assim:

SASS

.classe {

...

@media (min-width: breakpoint(pequeno)) {

...

}

}

Se não for o caso de mobile first, pode ser:

SASS

.classe {

...

@media (max-width: breakpoint(pequeno)) {

...

}

}

Este mixin cria uma caixa centralizada na tela com posicionamento absoluto (se o elemento pai tiver algum position definido vai centralizar com relação à ele). Ideal para criar uma tela de login, por exemplo. Não precisa usar em modais, pois este elemento já foi criado dentro do Front-Alego. Esta caixa não precisa ter tamanho fixo.

Sintaxe: @include centraliza-caixa($largura, $altura);

$largura
Define a largura da caixa. Este valor pode ser omitido.
$altura
Define a altura da caixa. Este valor pode ser omitido. Se for usar este valor, mas sem a altura, lembre-se de escrever o nome da variável.

Chamando o mixin sem os valores:

SASS

.sem-valor {

@include centraliza-caixa;

}

Chamando o mixin com os valores:

SASS

.com-medidas {

@include centraliza-caixa(35vw, 50vh);

}

Chamando o mixin com o valor de largura:

SASS

.so-largura {

@include centraliza-caixa(35vw);

}

Chamando o mixin com o valor de altura:

SASS

.so-altura {

@include centraliza-caixa($altura: 50vw);

}

Este mixin serve para criar itens de mesmo tamanho distribuídos ao longo de algum container. Ele deve ser aplicado no container (que se torna display: flex).

Sintaxe: @include grupo-flex($quantidade, $margem, $largura, $largura-minima);

$quantidade
A quantidade de caixas por linha. Sem valor padrão
$margem
Determina a margem entre os elementos. Se não quiser nenhuma margem, coloque o valor como 0. Valor padrão: 10px convertidos para rem.
$largura
Determina a largura de cada uma das caixas (elementos filhos). Este valor é opcional. Se omitido vai calcular a largura em porcentagem conforme a quantidade.
$largura-minima
Determina o tamanho mínimo de cada um dos elementos filhos. Especialmente se a largura comum (acima) estiver omitida e sair como porcentagem. Valor opcional.

Colocando 5 itens por linha:

SASS

.cinco-itens-linha {

@include grupo-flex(5);

}

Colocando 5 itens por linha, com margem de 1rem:

SASS

.cinco-itens-linha {

@include grupo-flex(5, 1rem);

}

Colocando 5 itens de 300px por linha:

SASS

.cinco-itens-linha {

@include grupo-flex(5, $largura: 300px);

}

Colocando 5 itens por linha com largura mínima de 300px cada:

SASS

.cinco-itens-linha {

@include grupo-flex(5, $largura-minima: 300px);

}

Este mixin altera a forma como o browser apresenta o texto quando o usuário seleciona o texto (experimente selecionar um parágrafo desta documentação, que usam os valores padrão).

Sintaxe: @include texto-selecionado($fundo, $cor-texto);

$fundo
Define a cor de fundo. Valor padrão: cor(fundo, escuro)
$texto
Define a cor do texto. Valor padrão: cor(texto, claro)

Como você pode notar, é possível aplicar o mixin sem valores e as cores mudam conforme as cores aplicadas nos temas de cor.

Este mixin pode ser aplicado em todo o site (no <body>) ou em elementos específicos (como um parágrafo, uma div e assim por diante).

Exemplo de uso no body sem valores:

SASS

body {

SASS

.urgente {

font-weight: bold;

color: cor-status(urgente);

@include texto-selecionado(cor-status(urgente-escuro), cor(texto, claro));

}

Faz listas que abrem e fecham mas que ainda exibem o título (como essa lista de mixins). O HTML deve incluir um checkbox no mesmo nível hierárquico do conteúdo que está ocultando. O conteúdo deve aparecer depois.

Este mixin deve ser aplicado dentro do seletor do checkbox.

Sintaxe: @include lista-oculta($classe);

$classe
A classe do conteúdo que vai ser exibido.

O HTML deve seguir o padrão:

  • Checkbox e conteúdo devem estar no mesmo nível hierárquico;
  • Depois do checkbox aparece o label (também no mesmo nível hierárquico);
  • Cada checkbox deve ter um id e este deve ser único. O label deve ter a propriedade for com o mesmo id do checkbox;
  • A classe do conteúdo não precisa ser diferente para cada item/lista (melhor que seja a mesma, inclusive, para deixar o CSS compilado menor);
  • O conteúdo vem aberto por padrão. Para ocultar o conteúdo daquele item, inclua checked="checked" no checkbox.

Exemplo de aplicação:

HTML

<div>

<input type="checkbox" class="check-lista" id="nome" />

<label for="nome">Título do item da lista</label>

<div class="conteudo-demo">

...

</div>

</div>
SASS

.check-lista {

@include lista-oculta(conteudo-demo);

}
