Loading

Todas as configurações básicas de tabela são encontradas no arquivo de configuração. Abaixo você vê todas as configurações.

Você pode chamar dois estilos de loading: com SVG e com CSS. O primeiro caso você chama um SVG disponibilizado no Front-Alego ou insere um no seu próprio sistema. Já o método CSS não depende de nenhuma imagem, somente do código (o método CSS é melhor, mas foi mantido o SVG por retrocompatibilidade, pois era a única forma disponível antes. O SVG tem a vantagem de personalizar a imagem, no entanto).

Para a escolha do loading, você deve escolher entre essas três classes:

.loading
Este loading ocupa todo o viewport e tampa a tela inteira. Use para quando você deseja impedir ação por parte do usuário no sistema.
.loading--bloco
Este loading ocupa todo o conteúdo de onde ele está. Use este para indicar que uma determinada parte do sistema está sendo carregado.
.loading--inline
Este loading aparece como inline e serve para indicar uma ação em determinado elemento, como por exemplo, um botão.

SVG

A imagem já está embutida na GEM. Caso você não esteja trabalhando em RAILS ou queira modificar a imagem, ela se encontra na pasta /images

Se quiser fazer uso de outra imagem animada para isso na GEM, basta colocar na pasta /assets/images

HTML

<div class="loading">

<img class="loading__img" src="<%= asset_path("loading.svg") %>" alt="Carregando">

<span class="loading__texto">Carregando</span>

</div>

CSS

O tipo da imagem do loading é determinado pelo $loading-tipo determinado no arquivo de configuração.

HTML

<div class="loading">

<span class="loading__img"></span>

<span class="loading__texto">Carregando</span>

</div>
